## 项目目标

搭建一个资源库（资源后台管理系统），用于汇集爬虫采集的数据，并支持后续批量分发给剪片环节。

## 硬指标（工程规范 · 不可妥协）

> 这是项目的强制约束，任何阶段都要遵守。

1. **Changelog**：每一次修改都必须有 changelog 记录（变更内容、原因、日期/版本）。
2. **Version control**：全程使用 git 进行版本控制，提交粒度清晰、信息可追溯。
3. **GitHub**：所有改动都要推送到 GitHub 仓库。

## 整体流程

1. **爬虫采集**：爬虫负责抓取数据（当前已有 `ig_downloader.py`，基于 instaloader 采集 Instagram 图片/Story）。
2. **接口入库**：爬虫采集到的数据，通过接口（API）写入资源库数据库。
3. **后台管理**：资源后台用于查看/管理入库的资源。
4. **批量分发**：在后台中安排，把资源批量发送出去，交给剪片环节。

## 已确认的关键决策（2026-06-05）

1. **批量分发方式**：通过 **接口** 发送。后台提供"批量发送"按钮，点击后调用下游接口把资源送去剪片。
   - 接口文档稍后由用户提供。
   - 当前阶段：先把"批量发送"按钮和接口对接位（stub / 预留封装）做好，文档到了再填实现。
2. **交付内容**：交给剪片下游的具体字段/格式，由稍后提供的接口文档定义。
3. **去重依据**：按 **文件内容哈希** 去重 —— 同一文件算重复（同一文件只入库一次）。
4. **资源状态机（四态）**：
   ```
   未处理 → 已发送切片 → 切片完毕 → 已发送到项目
   ```
   - 未处理：刚入库、还没挑选发送。
   - 已发送切片：已通过接口发出去给剪片。
   - 切片完毕：剪片完成（回执如何回填，待接口文档明确）。
   - 已发送到项目：成片已交付到对应项目。
